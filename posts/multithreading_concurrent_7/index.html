C++11并发与多线程(七)——std::async和std::future

2022-01-12
1 min read
C++

一、概念

  std::async是一个函数模板，用来启动一个异步任务，返回一个std::future对象，std::future是个类模板。

  异步任务：创建一个线程并开始执行对应线程的入口函数，返回一个std::future对象，它含有线程入口函数返回的结果，用std::future::get()来获取结果。

  std::future，也可以说是提供了一种访问异步操作结果的机制。std::future对象里会保存一个值，在将来（异步操作执行完之后）可以拿到。

二、用法

头文件

#include <future>

初始化

int fun() {
    return 123;
}
// ...
// 用一个可调用对象创建一个线程并开始执行，流程不会卡到这
// 尖括号里的类型就是函数的返回类型
std::future<int> res = std::async(fun);

get() （只能调用一次）

 // 如果运行到了get()而还没得到结果（fun()还没有return），就会阻塞到这一行直到拿到了结果
cout << res.get() << endl; // 123

wait()

 // 进入阻塞状态直到之前创建的线程走完
res.wait();

三、可选择传入的参数

std::launch::deferred

  表示线程入口函数调用被延迟到std::future的wait())或者get()函数调用时才执行。如果一直不调用，则不会创建线程。如果调用了，也不会创建子线程，线程入口函数会直接在主线程执行。

std::future<int> res = std::async(std::launch::deferred, fun);

std::launch::async

  表示线程会在调用async()函数时就创建（与不传入参数效果一致）。

std::future<int> res = std::async(std::launch::async, fun);
