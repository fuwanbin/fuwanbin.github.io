A.Park Lighting

给n×m个格子，格子之间那条线是路；你可以在路的中间放一个路灯，来照亮相邻两个格子，求照亮所有格子的最少路灯数。

int n, m;
void solve()
{
	cin >> n >> m;
	if (n % 2 == 0 || m % 2 == 0)
		cout << m * n / 2 << endl;
	else
		cout << m * (n - 1) / 2 + (m + 1) / 2 << endl;
}

B.Maria Breaks the Self-isolation

有1个人要邀请n个朋友来院子玩，但是第i个人要来的前提是当他到门口时要见到a[i]个人，包括同时到的人和主人。一次可以邀请多个人，且他们会同时到达，问最多能邀请多少人。

int n;
int arr[200000];
void solve()
{
	cin >> n;
	for (int i = 1; i <= n; i++)
		cin >> arr[i];
	sort(arr + 1, arr + n + 1);//先从要求少的人开始
	int now = 1;//已经到的人，包括主人
	int will = 0;//加入邀请名单的人，即将会有will个人同时到门口
	for (int i = 1; i <= n; i++) 
	{
		will++;//加入名单
		if (arr[i] <= now + will - 1)//已到的人加名单上的人已经符合要求，就可以邀请过来了,减1是剪掉了自己
		{
			now += will;
			will = 0;
		}
	}
	cout << now << endl;
}

C.Celex Update

这是我在比赛最后一分钟才想出来的，是最崩溃的。前两题17分钟做完，然后“挂机”103分钟。 给出这样一个表，给两个坐标，求从一个点到另一个点有多少个结果不同的方案，这里的结果指路径上所有数值之和。 天真的我以为每一条路径的结果都是不同的，就一直往这方面想。在最后大概十几分钟，才发现。我马上列了几个例子，很巧在最后一分钟找到了做法。 我们经过几个例子枚举了所有情况后，发现了他的规律。我们发现，从(x, y)出发，先往右走，再往下走，即最小结果minans。先往下走，再往右走，即为最大情况maxans。且在minans到maxans的所有结果，都可以走出来。所以答案为maxans-minans+1. 以3×6为例，显而易得，答案为1+2+2+2+2+1 + 1，即为2×5 + 1。 再多举两个例子，公式就出来了，ans=(xx-x)*(yy-y)+1;

ll x, xx, y, yy;
void solve()
{
	cin >> x >> y >> xx >> yy;
	cout << (yy - y) * (xx - x) + 1<< endl;
}
