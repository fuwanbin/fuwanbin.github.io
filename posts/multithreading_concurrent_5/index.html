C++11并发与多线程(五)——多线程下的单例模式

2022-01-11
2 min read
C++

一、单线程下的单例模式实现

class A {
private:
    A() {} //私有化构造函数

private:
    static A* instance;

public:
    static A* getInstance() {
        if (instance == nullptr) {
            instance = new A();
            //  由于gc是static的，所以能用于在整个程序结束时回收instance。
            static GC gc; 
        }
        return instance;
    }
    // 内部类，用来释放对象
    class GC {
    public:
        // 整个程序退出时调用，释放内存，instance置空防止悬空指针
        ~GC() {
            if (instance != nullptr) {
                delete instance;
                instance = nullptr;
            }
        }
    };
};
// static成员的初始化
A* A::instance = nullptr;
int main(void)
{
    A* a = A::getInstance(); 
    A* b = A::getInstance();
    return 0;
}

二、多线程下的单例模式问题分析

  我们给以上代码加点东西。

void mythread() {
    cout << "线程开始\n";
    A* a = A::getInstance();
    cout << "线程结束\n";
}
int main() {
    thread myt1(mythread);
    thread myt2(mythread);
    myt1.join();
    myt2.join();
}

  代码中存在一个问题，就是当两个线程同时走进getInstance()函数，同时判断instance是否为nullptr，得到的结果是，两个线程都能走进if里面，先后生成了2个A类对象。因此在判断instance是否为空之前，我们要用互斥量上锁。

std::mutex mymutex;
class A{
    // 略
public:
    static A* getInstance() {
        std::unique_lock<std::mutex> mylock(mymutex);
        if (instance == nullptr) {
            instance = new A();
            static GC gc;
        }
        return instance;
    }
    // 略
}

  这次，我们加入了互斥量，解决了会重复创建A类对象的问题，但是这还不够。我们知道，instance的初始化，在程序中只会发生一次，之后调用getInstance()返回new过的instance即可，其实根本不需要用到互斥量。而以上代码，为了第一次的初始化，牺牲了整个程序之后的效率，不是个很好的处理方式。我们使用双重检查改进代码。

static A* getInstance() {
    // 只有在第一次初始化会进入第一个if，只后会直接返回instance
    if (instance == nullptr) { // 双重检查
        std::unique_lock<std::mutex> mylock(mymutex);
        if (instance == nullptr) {  
            instance = new A();
            static GC gc;
        }
    }
    return instance;
}

  现在应该是没有其他的问题了。为了方便，C++11已经实现了一个函数来保证一个函数只会调用一次：std::call_once()。

三、std::call_once()

std::once_flag f; // 系统定义的标记
class A {
private:
    A() {} //私有化构造函数
    static void createInstance() { 
        // 执行程序，发现只会输出一次
        cout << "instance被创建";
        instance = new A();
        static GC gc;
    }
private:
    static A* instance;

public:
    static A* getInstance() {
        std::call_once(f, createInstance); //第二个参数写入只执行一次的函数名
        return instance;
    }
    //略
};
void mythread() {
    A* a = A::getInstance();
}
A* A::instance = nullptr;
int main(void)
{
    thread myt1(mythread);
    thread myt2(mythread);
    myt1.join();
    myt2.join();
    return 0;
}
